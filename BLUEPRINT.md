# 🏛 ARKHÉ STUDIO - BLUEPRINT & MANIFIESTO DE ARQUITECTURA (NIVEL DIOS)

## 👁️ LA VISIÓN
**ARKHÉ no es una app, no es un editor de texto ni un reproductor. Es una FÁBRICA DE ACTIVOS DE PROPIEDAD INTELECTUAL REGISTRABLES.**
Su propósito absoluto es convertir ideas efímeras (letras, melodías, audios) en **activos legales, verificables y monetizables**.

El arquitecto (usuario) interactúa con un sistema que automatiza:
- Fijación en un medio tangible.
- Generación de evidencia inmutable (Hash SHA-256, Timestamp).
- Empaquetado legal (Letras PDF, Certificados de Autoría, Data JSON).
- Separación técnica y jurídica entre **Obra Musical** (Letra/Melodía) y **Fonograma** (Grabación/Master).

## ⚙️ EL STACK TÉCNICO
El motor de ARKHÉ se sostiene en una arquitectura React full-stack.
- **Frontend:** React 18, Vite, Tailwind CSS para estilización rápida y fluida.
- **Visualización de Audio:** `wavesurfer.js` para renderizado espectral inmersivo.
- **Motor Criptográfico Legal:** `js-sha256` para inmutabilidad.
- **Documentos & Compiladores:** `jspdf` para certificados legales visuales, `jszip` para los empaquetados listos para distribución (INDAUTOR/Distribuidoras).
- **Infraestructura Evolutiva:** Motores "GPT Strategist" y "Automatic Evolution System" (Simulados/Integrados) para control de canibalización de contenido.

## 🛡️ LAS REGLAS DE ORO (Nunca romper estas condiciones)
1. **NO SOMOS UN EDITOR DE LETRAS:** Cada letra guardada DEBE procesarse en el `HashEngine`. No hay obras sin huella.
2. **FIJACIÓN LEGAL OBLIGATORIA:** Toda obra debe poder justificar: *Fijado en medio tangible + Identificable + Fecha verificable + Autoría irrefutable*.
3. **TRAZABILIDAD Y VERSIONES:** Nunca se sobrescribe. Los cambios se adjuntan como nuevas *versiones* con nuevos Hashes, manteniendo la reversibilidad.
4. **LOS ERRORES NO TUMBAN TODO (Degradación Elegante):** Si un módulo falla (ej. motor IA), el sistema de fijación legal base debe continuar intacto.
5. **ATRIBUCIÓN RIGUROSA:** Porcentajes de derechos (`Ownership`) controlados siempre que sumen 100%. No se mezclan autores, no se asume participación sin confirmación.
6. **TODO DEBE SER MONETIZABLE:** Si el archivo resultante no sirve para registrarse, licenciarse o venderse, no se programa en el sistema.
